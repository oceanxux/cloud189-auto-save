# 媒体服务器联动 (Emby/Jellyfin/Plex)

Cloud189 Auto Save 与主流媒体服务器深度联动，让云端资源像本地文件一样无缝呈现。

---

## 1. STRM 文件核心理念

**STRM** 是一种仅包含播放地址的小文件。通过本项目，Emby 不需要下载巨大的 4K 视频，只需读取集成了直链或代理地址的 `.strm` 文件即可流畅播放。

### 实时生成 (Auto-STRM)
系统在转存成功后，会自动在本地映射的 `strm` 文件夹（Docker 中的 `/home/strm`）生成对应的 `.strm` 文件及其目录结构。

### 懒转存 (Lazy STRM)
在网页端或 Bot 使用“懒转存”创建任务。系统会**立即**生成 STRM，但不会立即占用您的网盘空间。只有当您真的在 Emby 中点击“播放”时，系统才会瞬间执行转存并换取最新的天翼云直链。

---

## 2. Emby/Jellyfin 路径替换

由于 Docker 映射或网络挂载，云盘内的路径与媒体服务器识别的路径往往不一致。

### 配置步骤
1. 前往“账号管理” -> “账号设置” -> “Emby 路径替换”。
2. 输入格式：`云端路径:本地映射路径`。
   - **示例**：`/影视/动漫:/media/animation`
3. 作用：当转存任务完成后，系统发送通知或刷新请求时，会自动将路径中的 `/影视/动漫` 转换为 `/media/animation`，确保 Emby 能够精确定位到该文件并刷新其元数据。

---

## 3. 自动刷新库

您可以配置系统在转存任务完成后自动触发 Emby 的增量扫描。

### 配置 Emby API
1. 在 Emby “服务器设置” -> “API 密钥”中生成一个 Key。
2. 在 Cloud189 “系统设置” -> “Emby 配置”中填入 `Server URL` 和 `API Key`。

### 优势
- **即刻入库**：转存完成 1 分钟内，文件即可出现在 Emby 的“最新剧集”中。
- **低资源消耗**：系统仅请求刷新变动的文件夹，避免了全库扫描导致的高 CPU 占用。

---

## 4. 播放代理与直链

天翼云盘的直链通常只有几小时有效期。

### 流代理 (Stream Proxy)
- **原理**：STRM 中的链接不直接指向天翼云，而是指向本项目内置的代理端口（默认 `8097`）。
- **流程**：播放器 -> 本项目 -> 动态换取最新直链 -> 重定向至天翼云。
- **配置**：在“系统设置”中开启“启用流代理”，并确保 `8097` 端口已在 Docker 中正确映射。

---

## 💡 提示
- **刮削建议**：开启“自动刮削”并配置 TMDB API 后，系统会自动在 STRM 旁生成 `.nfo` 和海报，让 Emby 刮削快如闪电。
- **权限管理**：确保 Docker 中的 `PUID/PGID` 与宿主机的媒体文件所有者一致，以保证读写正常。
